Open\+GL permet le rendu de scène 3D dans un bitmap. Si l’environnement 3D contient des données ~\newline
de grande taille, il est difficile voire impossible d’en effectuer le rendu (transmission sur la carte ~\newline
graphique, mémoire insuffisante sur celle-\/ci). La proposition de ce projet est donc de diviser la ~\newline
scène en complexité, en répartissant les données 3D sur plusieurs fenêtres, de manière synchrone. ~\newline
Tous les rendus doivent donc être synchronisés sur les interactions (clavier ou souris pour faire ~\newline
tourner l’objet, ou zoomer). La 2e étape de ce projet sera d’effectuer une distribution de données 3D ~\newline
sur des machines reliés en réseau, via \mbox{\hyperlink{classOpenMPI}{Open\+M\+PI}} et d’en faire le rendu sur chaque machine. ~\newline
L’application devra être développée en C++, avec Open\+Mesh pour la lecture de fichier 3D, Open\+GL ~\newline
pour le rendu, S\+DL pour le (multi-\/) fenêtrage et les interactions, \mbox{\hyperlink{classOpenMPI}{Open\+M\+PI}} pour la distribution sur ~\newline
plusieurs machines d’un réseau local et Doxygen pour la documentation des fonctions et classes. ~\newline
Les temps de lecture de fichier, division des données, échange réeau et rendu devront être mesurés ~\newline
et comparés sur plusieurs objets ou sur plusieurs définitions de surface. ~\newline
